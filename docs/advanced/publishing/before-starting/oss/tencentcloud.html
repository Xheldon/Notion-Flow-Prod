腾讯云 COS

对象存储（Cloud Object Storage，COS）是由腾讯云推出的无目录层次结构、无数据格式限制，可容纳海量数据且支持 HTTP/HTTPS 协议访问的分布式存储服务。腾讯云 COS 的存储桶空间无容量上限，无需分区管理，适用于 CDN 数据分发、数据万象处理或大数据计算与分析的数据湖等多种场景。官网地址见 腾讯云 COS。

提示

如果你已经有了一个用来放博客图片的存储桶，那么可以跳过这一节，直接到后面的 获取存储桶和区域 一节。如果你没有腾讯云 OSS 服务，你需要先 购买，我的配置是 中国大陆通用、按存储容量计费、标准存储、10G、1年、加购数据处理资源包类型为「流量-100G/年」，总费用 20.12 元/年。

创建存储桶

进入腾讯云控制台 COS 存储桶页面，点击创建存储桶，访问权限默认设置「私有读写」即可，后续会配置 CDN 对其的访问。其他配置根据你的实际情况设置：

之后一路下一步，然后点击「创建」即可。

获取存储桶和区域

存储桶（Bucket）和区域（Region）是 COS 中的两个重要概念，对象存储是以「桶」为单位进行隔离的。

进入 COS 存储桶页面，在下方列表中找到你刚刚新建的存储桶，即可获得存储桶名称和区域：

图片中，notion-blog-1254272402 即为存储桶名称，ap-nanjing 即为区域名称，记下这两个信息，后面在插件配置的时候会用到。

获取 SecretId 和 SecretKey

进入腾讯云 秘钥创建界面，点击新建密钥，然后记录下弹出框中的 SecretId 和 SecretKey 即可。

注意

注意，这个 SecretKey 只会出现一次，请及时保存。

提示

如果你是使用主账号（就是默认账号）登陆，那么此时你创建的是权限最高的 SecretId 和 SecretKey，如果泄露会有一定的安全风险。

如果你想要更安全的配置，可以使用子账号来授权，需要 COS 服务对子账号授权读写权限，具体请查看腾讯云文档。
